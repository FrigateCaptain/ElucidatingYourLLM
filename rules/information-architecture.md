# Information Architecture — Архитектура информации и качество контента

---

## Information Architecture [MANDATORY]

### Принцип
Устранять неоднозначности, дубликаты, несоответствия и неясные источники.

### Система канонических определений [CANONICAL]

**Правило**: Каждое утверждение должно иметь прослеживаемый источник.

- **Single Source of Truth**: Каждое понятие имеет ОДНУ каноническую версию
- **Метка [CANONICAL]**: Ставить в документе-источнике, где информация впервые установлена
- **Ссылки [REF:]**: Все остальные упоминания используют `[REF: папка/файл.md#раздел]`
- **Точное повторение**: Использовать идентичные формулировки при ссылках
- **Явная адаптация**: При изменениях помечать `[ADAPTED from файл.md: "оригинал" → "адаптация"]`

### Метки источников

- **[CANONICAL]** — авторитетное определение
- **[CONFIRMED: источник]** — подтверждённая информация
- **[PLACEHOLDER: владелец]** — информация для заполнения
- **[REF: путь]** — перекрёстная ссылка

### Требования отслеживания

- **No Orphan Statements**: Каждое утверждение должно быть прослеживаемо к источнику
- **Clear Attribution**: Указывать откуда информация — записи, исследования, расчёты

### Anti-Hallucination Rules

- **Удалять неподтверждённый контент** — только информация из надёжных источников
- **Использовать реальные данные** — конкретные метрики из записей
- **Источник для каждого утверждения** — `[CANONICAL]` или `[REF:]`
- **Помечать плейсхолдеры** — `[PLACEHOLDER: владелец]`
- **Датировать всё** — включать timestamps "Последнее обновление"
- **Поддерживать отслеживаемость** — при обновлении обновлять все ссылки

## Content Quality Standards [MANDATORY]

1. **Удалять галлюцинированный контент** — только подтверждённая информация
2. **Использовать реальные данные** — метрики из реальных операций
3. **Источник для каждого утверждения** — `[CANONICAL]` или `[REF:]`
4. **Помечать плейсхолдеры** — `[PLACEHOLDER: владелец]`
5. **Перекрёстные ссылки** — связывать документы через `[REF:]`
6. **Датировать всё** — timestamps "Последнее обновление"
7. **Поддерживать отслеживаемость** — обновлять ссылки при изменении источников
