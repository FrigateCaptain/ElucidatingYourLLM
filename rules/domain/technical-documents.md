# Technical Documents — Оформление документов в technical/

**Доменное правило.** Применять при создании/редактировании .md в `technical/`.

Globs: `**/technical/**/*.md`

---

## Назначение документа

Каждый документ в `technical/` должен содержать в начале (после заголовка) поле **«Назначение документа»** — одно-два предложения: зачем существует и кому полезен.

## Структура документа

### Заголовок
- Первый уровень: `# Название документа`
- Понятное, описательное название

### Оглавление (обязательный элемент)
Сразу после заголовка:
```markdown
## Содержание

- [Название раздела 1](#якорь-ссылки)
- [Название раздела 2](#якорь-ссылки-2)
```
Якоря: все строчные, пробелы → дефисы, спецсимволы удаляются. После оглавления — `---`.

### Разделы
- `##` — основные разделы
- `###` — подразделы

## Форматирование
- **Жирный** для терминов, *курсив* для акцентов
- Блоки кода с указанием языка
- Ссылки внешние: `[текст](URL)`, внутренние: `[название](../technical/файл.md)`

## Именование файлов
- Описательные имена, подчёркивания вместо пробелов
- Технические термины: `про_язык_markdown.md`

## Даты (обязательный элемент)
В конце документа:
```markdown
---

**Дата создания:** YYYY-MM-DD
**Последнее обновление:** YYYY-MM-DD
```

## Чеклист перед сохранением
- Есть «Назначение документа»
- Есть оглавление с рабочими якорями
- Указаны даты создания и обновления
- Структура разделов логична
