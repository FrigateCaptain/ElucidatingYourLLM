# Markdown Dates — Даты создания и актуализации

**Доменное правило.** Применять при работе с README.md и файлами-обзорами.

Globs: `**/*.md`

---

## Markdown Files Date Rule [MANDATORY]

Все README.md и файлы-обзоры должны содержать дату создания и дату актуализации в конце файла.

**Формат:**
```markdown
---

*Дата создания: [день] [месяц] [год], [часы]:[минуты]*
*Дата актуализации: [день] [месяц] [год], [часы]:[минуты]*
```

**Правила:**
1. **При создании нового файла**: обе даты одинаковые — момент создания
2. **При обновлении**: менять только «Дата актуализации». «Дата создания» — не трогать
3. **Формат даты**: «5 января 2025, 17:45» (день месяц год на русском, 24-часовой формат)

**Область применения:** README.md и файлы-обзоры во всех рабочих папках проекта.
