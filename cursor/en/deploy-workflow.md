# Deploy Workflow: Cursor Rules

*Этот файл — для AI-ассистента пользователя. AI читает его и выполняет деплой правил из ElucidatingYourLLM.*

## Контекст

Ты помогаешь пользователю установить правила Cursor из репозитория ElucidatingYourLLM.
Правила находятся рядом с этим файлом в папке `cursor/ru/`:

- `.cursorrules` — 11 основных правил (безусловная загрузка)
- `.cursor/rules/*.mdc` — 3 правил, запускаемых по условию:
  - `file-editing.mdc`
  - `pre-install-check.mdc`
  - `structural-analogy.mdc`

## Шаги деплоя

### 1. Прочитай правила из этой папки

- Прочитай `.cursorrules` из папки `cursor/ru/` этого репозитория.
- Составь список файлов `.cursor/rules/*.mdc` в папке `cursor/ru/`.

### 2. Проверь текущие правила пользователя

- Найди `.cursorrules` в корне рабочего проекта пользователя (если есть).
- Найди файлы в `.cursor/rules/` пользователя (если есть).
- Если файлов нет — пропусти бэкап и диагностику, сразу переходи к шагу 5.

### 3. Бэкап

Если у пользователя есть существующие файлы — спроси подтверждение и сделай бэкап:

```bash
cp .cursorrules .cursorrules.backup
cp -r .cursor/rules .cursor/rules.backup
```

### 4. Диагностика (если у пользователя уже есть правила)

Сравни и покажи пользователю три категории:

- **Новые** — правила из ElucidatingYourLLM, которых нет у пользователя. Можно добавить безопасно.
- **Перекрывающиеся** — аналогичная тема уже покрыта у пользователя. Покажи разницу и объясни, что меняется.
- **Конфликтующие** — явное противоречие. Предложи варианты: взять новое, оставить старое, объединить.

### 5. Установка

После подтверждения пользователя:

- Скопируй `.cursorrules` из `cursor/ru/` в корень рабочего проекта пользователя.
- Скопируй все `.mdc` файлы из `cursor/ru/.cursor/rules/` в `.cursor/rules/` проекта.

Создай `.cursor/rules/` если не существует.

### 6. Отчёт

Сообщи что установлено:
- 11 основных правил в `.cursorrules`
- 3 правил в `.cursor/rules/`
- Перечисли файлы из `.cursor/rules/`, которые были установлены.
