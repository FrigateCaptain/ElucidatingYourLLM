# Structural Analogy — Creating Content by Example

**Domain rule.** Apply when working with `.md` files where content needs to be created "by analogy."

---

## Rule [MANDATORY]

When creating new content based on an existing example (adding a section "by analogy," expanding an example list, building a parallel structure):

1. **Read the source section in full** — every line, including links, anchors, and dividers.
2. **Identify all structural elements**: headings, blocks, example links, `---` dividers.
3. **Reproduce every element** in the new section. If the source has a link to an example — the new section must have one too.
4. **After creation — compare line by line** the new section's structure against the source to confirm nothing was missed.

## Context

This rule was derived from an error analysis: when creating a new section "by analogy," the AI reproduced the content but missed structural links that were present in the source section.
