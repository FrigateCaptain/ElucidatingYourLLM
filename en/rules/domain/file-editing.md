# File Editing — Literal Execution and File Protection

**Domain rule.** Apply to any file editing work.

---

## Literal Execution of Requests [MANDATORY]

- Make **exactly** the changes the user requests.
- **DO NOT make** unrequested changes (column order, formatting, renaming, restructuring, etc.).
- If the AI believes an additional change would be useful — **ask for confirmation first**, describing what and why.
- Only after explicit user confirmation — make the additional change.

**Exception — scripts**: when creating or modifying executable scripts (`.py`, `.sh`, `.bash`) — always apply the full Scripts Management process. Updating spec, COMPLIANT, REGISTRY is a mandatory part of any change.

## File Protection [MANDATORY]

- Work only with explicitly specified files and sources. Do not touch other files, even if they seem similar or contextually related.
- If there are multiple potential sources (ODT, TXT, MD, etc.) or any doubt remains — pause and clarify with the user which source to use and what exactly to change.
- Do not alter the meaning, structure, or format of a document without a direct request.

## Backups and Confirmations [MANDATORY]

- Before experiments or bulk edits — make a copy/backup of the original file and work in the copy.
- Before applying changes — describe what is planned and wait for confirmation. Afterwards — briefly note what was changed and which files were affected.
