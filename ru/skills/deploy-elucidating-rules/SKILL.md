# SKILL: deploy-elucidating-rules

## Назначение

Интерактивный деплой правил из репозитория ElucidatingYourLLM в Cursor или Claude Code. Задаёт вопросы по выбору платформы, правил и настройкам, затем создаёт и размещает файлы.

## Когда использовать

Когда пользователь хочет:
- Задеплоить правила из ElucidatingYourLLM в свой Cursor или Claude Code
- Обновить существующие правила до новой версии
- Добавить отдельные доменные правила

## Пошаговые инструкции

### Шаг 1. Определить платформу

Спросить пользователя:
> «Куда деплоим правила?
> 1. Cursor (`.cursorrules` + `.cursor/rules/`)
> 2. Claude Code (`CLAUDE.md` / `~/.claude/CLAUDE.md`)»

### Шаг 2. Определить workspace

**Для Cursor:**
> «В какой workspace? Укажите путь к папке проекта, или "глобально" для `~/.cursor/`»

**Для Claude Code:**
> «Глобально (`~/.claude/CLAUDE.md`) или для конкретного проекта? Если для проекта — укажите путь.»

### Шаг 3. Выбор базовых правил

Показать список и спросить:
> «Какие базовые правила включить?
> - [x] core.md (поведение AI, защита от ошибок) — рекомендуется
> - [x] information-architecture.md (информационная архитектура) — рекомендуется
> - [x] file-management.md (управление файлами) — рекомендуется
> - [ ] scripts-management.md (управление скриптами) — нужен если пишете скрипты регулярно
> - [x] git-workflow.md (git) — рекомендуется»

### Шаг 4. Выбор доменных правил

> «Какие доменные правила нужны? Отметьте нужные:
> - [ ] python-proxy.md — для работы с Python через прокси/VPN: корректная передача прокси во все HTTP-библиотеки
> - [ ] pre-install-check.md — для установки ПО с помощью AI: проверка совместимости с системой перед установкой (рекомендуется всем)
> - [ ] markdown-dates.md — для ведения документации в markdown: автоматическое проставление дат создания и актуализации
> - [ ] excel-files.md — для работы с Excel через Python: стандарты чтения, записи и оформления файлов
> - [ ] generated-docs.md — для создания DOCX/ODT программно: стандарты структуры и форматирования генерируемых документов
> - [ ] video-editing.md — для редактирования видео через ffmpeg: готовые паттерны команд, обработка ошибок
> - [ ] technical-documents.md — для ведения технической документации в папке `technical/`: структура и требования к оформлению
> - [ ] file-editing.md — для работы с файлами через AI: буквальное выполнение запросов, защита от незапрошенных изменений, бэкапы (рекомендуется всем)
> - [ ] settings-change-tracking.md — для изменения настроек с помощью AI: автоматическая фиксация всех изменений в документации (рекомендуется всем)
> - [ ] backlogs-management.md — для ведения бэклогов задач: структура файлов, реестр, порядок работы
> - [ ] odt-fonts.md — для создания ODT-документов: стандарты шрифтов и оформления (LibreOffice)
> - [ ] projects-management.md — для управления проектами: реестр, бэклог, подпроекты, именование
> - [ ] rules-management.md — для создания и управления правилами AI-ассистента: размещение, структура, реестр, валидация
> - [ ] skills-management.md — для работы со skills AI-ассистента: безопасность, аппрув, создание
> - [ ] structural-analogy.md — для создания контента по аналогии с образцом: сохранение структуры, ссылок, разделителей»

### Шаг 5. Язык правил

> «Правила написаны на русском. Оставить на русском или адаптировать на другой язык?»

### Шаг 6. Структура для Cursor

**Только для Cursor:** спросить про структуру:
> «Как организовать правила:
> 1. Монолитно — всё в одном `.cursorrules` (проще)
> 2. Модульно — базовые в `.cursorrules`, доменные в `.cursor/rules/*.mdc` (рекомендуется)»

### Шаг 7. Создать файлы

На основе ответов:

1. Проверить наличие папки `.cursor/rules/` (для модульного варианта Cursor)
2. Для каждого выбранного файла из `ru/rules/` (или `en/rules/`): прочитать содержимое, **удалить строки `*Дата создания:*` и `*Дата актуализации:*`** (они служебные, не должны попасть в правила пользователя)
3. Создать основной файл правил (`.cursorrules` или `CLAUDE.md`) из очищенного содержимого
4. Создать доменные `.mdc` файлы с frontmatter (для модульного Cursor) из очищенного содержимого
5. Доложить пользователю: какие файлы созданы и куда

### Шаг 8. Проверка

Предложить пользователю проверить:
> «Файлы созданы. Рекомендую проверить:
> 1. Начните новый разговор в Cursor/Claude Code
> 2. Спросите: "Какие правила у тебя есть?"
> 3. AI должен воспроизвести ключевые принципы»

### Шаг 9. Персонализация

Предложить следующий шаг:
> «Правила задеплоены. Хотите сейчас персонализировать — убрать лишние разделы, изменить языковую политику или добавить специфичные для вашего проекта правила?»

## Примечания

- При обновлении до новой версии: сначала показать CHANGELOG.md, затем спросить, какие изменения применить
- Если правила уже существуют — предупредить и создать бэкап (суффикс `.backup`)
- Путь к репозиторию ElucidatingYourLLM уточнить у пользователя или определить автоматически (текущая папка, если открыт репозиторий)
