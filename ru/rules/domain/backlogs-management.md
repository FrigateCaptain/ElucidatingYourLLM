# Backlogs Management — Управление бэклогами

**Доменное правило.** Применять при создании и ведении бэклогов задач.

---

## Расположение

- Хранить бэклоги в выделенной папке проекта (например, `backlogs/`)
- Вести реестр бэклогов: единый список всех бэклогов с указанием области и количества открытых задач

## Создание нового бэклога

1. Создать файл `{название}_backlog.md` (название на английском, snake_case)
2. Структура файла:
   - Заголовок и краткое описание области (1 предложение)
   - Раздел `## Открытые задачи` — чекбокс-список `- [ ]`
   - Раздел `## Выполненные задачи` — перенесённые из открытых, с `- [x]`
   - Даты создания и актуализации в конце
3. Добавить запись в реестр бэклогов
4. Обновить дату актуализации реестра

## Формат задачи

```markdown
- [ ] **Краткое название задачи**
  Описание: что нужно сделать и зачем (1-3 предложения).
  Ссылка на план/контекст: [путь/к/файлу.md]
```

- Каждая задача — один чекбокс с жирным заголовком
- Описание — на следующей строке с отступом (2 пробела)

## Работа с задачами

**Добавление:** добавить в «Открытые задачи», обновить счётчик в реестре и дату актуализации.

**Завершение:** перенести в «Выполненные», изменить `- [ ]` на `- [x]`, добавить дату: `*(выполнено: 9 марта 2026)*`, обновить счётчик.

**Отмена:** перенести в «Выполненные» с пометкой `*(отменено: дата — причина)*`.

## Когда создавать новый бэклог

- Область задач **не пересекается** с существующими бэклогами
- Задачи имеют **общую тему**, которую можно описать одним предложением
- Если в существующем бэклоге задача выглядит инородно — это сигнал к созданию нового
