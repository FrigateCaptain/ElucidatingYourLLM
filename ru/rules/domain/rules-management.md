# Rules Management — Управление правилами AI-ассистента

**Доменное правило.** Применять при создании, редактировании или удалении файлов правил.

---

## Размещение

- **Базовые правила** (нужны в каждом разговоре: поведение AI, безопасность, git, язык, формат ответов) — в основном файле правил (`.cursorrules`, `CLAUDE.md` или аналоге платформы).
- **Доменные правила** (для конкретных типов файлов, инструментов, форматов) — в отдельных файлах с указанием области применения (globs или description).
- Правила в обычных `.md`-файлах без специального подключения платформа **не подгружает автоматически**.

## Структура правила

Каждый файл правила начинается с метаданных (frontmatter или заголовка):
- Описание: для чего применяется
- Область применения: glob-паттерн файлов или семантическое описание
- Флаг `alwaysApply`: только если правило нужно в **каждом** разговоре

## Выбор типа активации

- **`alwaysApply: true`** — только если правило нужно в КАЖДОМ разговоре. Тест: «Если пользователь спросит "что такое 2+2?" — нужно ли это правило в контексте?» Если нет — использовать globs.
- **`globs`** — предпочтительный вариант для доменных правил. Указывать конкретные паттерны файлов.
- **`description` без globs** — для правил, которые сложно привязать к файлам; платформа подгрузит по семантической релевантности.

## Принципы качества [MANDATORY]

- **Одно правило = одна цель.** Если правило содержит несвязанные темы — разделить на отдельные файлы.
- **Не дублировать.** Каждое правило существует в одном месте. При обнаружении дубликата — объединить.
- **Контроль размера.** Хороший ориентир — до 50 строк. Если правило превышает ~500 строк — предложить пользователю дробление.
- **Разделять правила и артефакты:**
  - **Правило** — условное предписание для классов ситуаций: «если [ситуация] — делать [X]».
  - **Skill** — пакет экспертизы для конкретной задачи, загружается по необходимости.
  - **Аналитический документ / разбор кейса** — в профильной папке, не в правилах.

## Реестр правил

Все правила регистрировать в едином реестре. При создании, изменении или удалении правила — обновить реестр.

## Валидация перед сохранением

- Имя файла — описательное, kebab-case
- Метаданные содержат `description`, `globs` или `alwaysApply`
- `alwaysApply: true` обоснован (тест «2+2»)
- Globs не слишком широкие
- Содержание — условные предписания, не инструкции/workflows
- Нет дублирования с другими правилами
- Если > 500 строк — предложить дробление пользователю
- Правило внесено в реестр
