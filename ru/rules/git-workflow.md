# Git Workflow — Правила работы с Git

---

## Git Workflow [MANDATORY]

**Для AI-ассистента:**
- **НИКОГДА не делать git push** без явного подтверждения пользователя
- **НИКОГДА не коммитить изменения** пока пользователь не укажет, что работа завершена
- **НИКОГДА не предполагать email пользователя** — спрашивать явно
- **НИКОГДА не редактировать главный файл правил** без явного запроса
- При смене фокуса пользователя — предлагать закоммитить предыдущую работу
- Предлагать коммиты когда готов набор изменений
- Ждать одобрения перед push

## Реестр репозиториев [CANONICAL]

Вести отдельный файл-реестр всех репозиториев проекта с указанием: локального пути, remote URL, ветки, статуса автосинхронизации.

**Сверяться с реестром** при любых операциях с remote (push, pull, clone, set-url).  
**Обновлять реестр** при: подключении нового репозитория, переименовании, смене remote URL, изменении автосинхронизации.

## Подключение нового репозитория

1. Инициализировать git: `git init`
2. Добавить remote через SSH: `git remote add origin git@github.com:<username>/<repo>.git`
3. Создать `.gitignore` (исключить: `*.log`, служебные папки)
4. Первый коммит и push: `git add . && git commit -m "..." && git push -u origin main`
5. Настроить автосинхронизацию при необходимости (systemd user services)
6. **Обновить реестр репозиториев**
