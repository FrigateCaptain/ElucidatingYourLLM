# Деплой в OpenClaw

Пошаговая инструкция по интеграции правил в workspace агента OpenClaw.

---

## Архитектура OpenClaw workspace

OpenClaw помещает в workspace агента набор `.md`-файлов. Агент читает их при каждом старте сессии.

Ключевые файлы workspace:
```
~/.openclaw/workspace/
├── AGENTS.md      ← главный инструктаж (порядок работы, протоколы)
├── SOUL.md        ← принципы и характер агента
├── IDENTITY.md    ← имя, стиль, эмодзи
├── USER.md        ← профиль пользователя
├── TOOLS.md       ← локальные настройки инструментов
└── memory/
    └── YYYY-MM-DD.md
```

## Шаг 1. Ответьте на вопросы

### 1.1 Что адаптировать

**Вопрос:** Что вы хотите перенести в OpenClaw:
- Только правила поведения (core) → идут в `AGENTS.md` / `SOUL.md`
- Правила работы со скриптами → идут в `AGENTS.md` (раздел о скриптах)
- Доменные правила → только если агент выполняет соответствующие задачи

### 1.2 Специфика агента на VPS

Некоторые правила не применимы к агенту на VPS:
- Правила GUI-инструментов — не применимы
- Правила про локальные пути ноутбука — требуют адаптации
- SSH-правила — применимы в обратную сторону (агент сам является удалённым сервером)

## Шаг 3. Адаптация под sandbox-модель

OpenClaw использует sandbox-изоляцию: каждая сессия получает копию workspace.

При адаптации правил:
- Пути: заменить абсолютные пути ноутбука на пути VPS
- Инструменты: убрать GUI-специфичные правила
- Память: добавить правила по использованию `memory/YYYY-MM-DD.md`

## Шаг 3а. Адаптация platform-specific правил

Если на шаге 1.1 выбраны `rules-management` и/или `skills-management` — эти правила содержат термины, специфичные для Cursor (`.mdc`, `globs`, `.cursor/rules/`). При деплое на OpenClaw выполните адаптацию:

### Таблица соответствия концепций

| Концепция в правиле | В Cursor | В OpenClaw |
|---------------------|----------|------------|
| Основной файл правил | `.cursorrules` | `AGENTS.md` / `SOUL.md` |
| Доменные правила | `.cursor/rules/*.mdc` с globs | Секции в `AGENTS.md` или отдельные `.md`-файлы в workspace |
| Метаданные правила | YAML frontmatter (`description`, `globs`, `alwaysApply`) | Заголовок секции |
| Активация по типу файла | `globs` | Нет (всё загружается всегда) |
| Реестр правил | `RULES_REGISTRY.md` | Оглавление workspace-файлов |
| Skills | `.cursor/skills/<name>/SKILL.md` | Инструкции и процедуры в `AGENTS.md` или `TOOLS.md` |
| Безопасность skills | Только локальные, только после явного аппрува | Только после явного аппрува |

### Порядок адаптации `rules-management`

1. Прочитать `ru/rules/domain/rules-management.md`
2. Заменить термины по таблице выше
3. Удалить раздел «Выбор типа активации» (нет globs/alwaysApply в OpenClaw)
4. Сохранить без изменений: одно правило = одна цель, не дублировать, контроль размера, реестр, валидация
5. Результат — секция «Rules Management» в `AGENTS.md`

### Порядок адаптации `skills-management`

1. Прочитать `ru/rules/domain/skills-management.md`
2. Заменить «`.cursor/skills/<name>/SKILL.md`» → «инструкции в `AGENTS.md` или `TOOLS.md`»
3. Удалить упоминание «Глобальные skills `~/.cursor/skills/`» (нет аналога)
4. Сохранить без изменений: только локальные, только после явного аппрува, не из внешних источников
5. Результат — секция «Skills Management» в `AGENTS.md`

## Шаг 4. Редактирование SOUL.md

> **Примечание**: при копировании содержимого из `ru/rules/`-файлов — **удалите строки `*Дата создания:*` и `*Дата актуализации:*`** в конце каждого файла. Они служебные и не должны попадать в workspace агента.

Пример блока для добавления в `SOUL.md`:

```markdown
## Принципы поведения

[скопировать содержимое ru/rules/core.md, разделы "AI Assistant Behavior", "Self-Reflection"]

## Защита от ошибок

[скопировать содержимое ru/rules/core.md, раздел "Technical Claims Verification"]
```

## Шаг 5. Редактирование AGENTS.md

Добавьте в `AGENTS.md` раздел с рабочими правилами. Пример структуры:

```markdown
## Rules

### Behavior
[из SOUL.md — краткие ссылки]

### Scripts
[из ru/rules/scripts-management.md — адаптированный вариант]

### Git
[из ru/rules/git-workflow.md]
```

## Обновление

При выходе новой версии правил:
1. `git pull` в папке ElucidatingYourLLM
2. Проверить CHANGELOG.md
3. Вручную внести изменения в `SOUL.md` и `AGENTS.md` на VPS

> **Примечание**: этот воркфлоу описывает только деплой *правил* в агента OpenClaw. Поддержка и мониторинг VPS (бэкап, мониторинг, скрипты, инциденты) — отдельный процесс.
